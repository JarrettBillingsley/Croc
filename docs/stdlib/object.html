module object

The object library provides access to aspects of the object model not covered by Croc's syntax. This includes adding and removing fields and methods of classes and reflecting over the members of classes and instances.

global function newClass(name: string, vararg)

A function for creating a class from a name and optional base classes. The built-in class declaration syntax in Croc doesn't allow you to parameterize the name, so this function allows you to do so.

The varargs are the optional base classes, and they must all be of type class. Of course, you can pass zero base classes.

global function fieldsOf(value: class|instance)

Given a class or instance, returns an iterator function which iterates over all the fields in that object.

The iterator gives two indices: the name of the field, then its value. For example:

class Base
{
	x = 5
	y = 10
}

class Derived : Base
{
	override x = 20
	z = 30
}

foreach(name, val; object.fieldsOf(Derived))
	writefln("{} = {}", name, val)
This will print out (though not necessarily in this order):

x = 20
y = 10
z = 30

Params:
value

is the class or instance whose fields you want to iterate over.

Returns:

an iterator function.

global function methodsOf(value: class|instance)

Just like fieldsOf, but iterates over methods instead.

Params:
value

is the class or instance whose methods you want to iterate over. If you pass an instance, it will just iterate over its class's methods instead (since instances don't store methods).

Returns:

an iterator function.

global function addMethod(cls: class, name: string, value)

Adds a new method to a class. The method can be any type, not just functions.

The class must not be frozen, and no method or field of the same name may exist.

global function addField(cls: class, name: string, value = null)

Adds a new field to a class.

The class must not be frozen, and no method or field of the same name may exist. The value parameter is optional.

global function addMethodOverride(cls: class, name: string, value)

Adds a new method to a class, overriding any existing method of the same name. Works just like the override keyword in an actual class declaration.

The class must not be frozen, and a method of the same name must exist.

global function addFieldOverride(cls: class, name: string, value = null)

Adds a new field to a class, overriding any existing field of the same name. Works just like the override keyword in an actual class declaration.

The class must not be frozen, and a field of the same name must exist. The value parameter is optional.

global function removeMember(cls: class, name: string)

Removes a member (method or field) from a class.

The class must not be frozen, and there must be a member of the given name to remove.

global function freeze(cls: class)

Forces a class to be frozen.

Normally classes are frozen when they are instantiated or derived from, but you can force them to be frozen with this function to prevent any tampering.

global function isFrozen(cls: class)

Returns:

a bool indicating whether or not the given class is frozen.

global function isFinalizable(value: class|instance)

Returns:

a bool indicating whether or not the given class or instance is finalizable.

If value is an unfrozen class, it will be frozen! There is no way to be sure a class is finalizable without freezing it first.

global function instanceOf(value, cls: class)

Returns:

true if value is an instance and it is an instance of cls, and false otherwise. In other words, it returns isInstance(value) && value.super is cls.
