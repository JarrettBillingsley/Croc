module file

This module provides access to the host file system. With it you can read, write, create, and delete files and directories.

You should use paths with forward slashes ('/') to separate the directories for the best cross-platform compatibility. This library will translate forward slash paths to backslash paths internally on Windows, but even on Windows it will always return forward slash paths.

global function inFile(name: string)

Open an existing file for reading.

Params:
name

is the name of the file to open.

Returns:

a stream.NativeStream object which represents the file. It will be readable, seekable, and closable. The file is completely unbuffered!

Throws:
OSException

if the file could not be opened for some reason.

global function outFile(name: string, mode: string = "c")

Opens a file for writing.

Params:
name

is the name of the file to open.

mode

describes how the file should be opened. It must be a one-character string, and it must be one of the following values:

"e"

means the specified file must already exist, and an exception will be thrown if not. If the file was opened successfully, the contents will not be modified, and the file position will be at the beginning.

"x"

is the opposite of "e": the file must not already exist, and a new file will be created.

"c"

will create the file if it doesn't exist, or if it does exist, it will open it and truncate its contents. This is the default mode if you don't pass anything for the mode parameter.

"a"

("append") will create the file if it doesn't exist, or if it does exist, it will open it and then seek to the end of the file, leaving the existing contents intact. Unlike the C stdio "a" mode or the POSIX O_APPEND mode, this will allow you to seek freely and writes will not always go to the end of the file.

Returns:

a stream.NativeStream object which represents the file. It will be writable, seekable, and closable. The file is completely unbuffered!

Throws:
OSException

if the file could not be opened for some reason.

global function inoutFile(name: string, mode: string = "e")

Just like outFile, except the file will also be readable. Furthermore, the mode defaults to "e".

global function readTextFile(name: string, encoding: string = ?utf-8, errors: string = strict)

Reads the contents of a text file, decoding it according to the given encoding, and returns the entire file contents as a string.

Params:
name

is the name of the file to read.

encoding

can be one of three things:

  • The name of an encoding as used by the text library.

  • The string '?', which means that the text file must start with a BOM (or UTF-8 signature "BOM"). The encoding will be determined automatically using text.detectBOM. If the file does not start with a BOM, an exception will be thrown.

  • A string starting with '?' followed by an encoding name, such as "?utf-8" (the default for this parameter). In this case, the encoding will be determined like with '?', but if the file does not start with a BOM, it will fall back to the encoding given. So "?utf-8" means "detect a BOM, but if there is none, decode it as UTF-8."

errors

is how malformed text should be read, as specified by the text library.

Returns:

the decoded contents of the file.

global function truncate(file: instance)

Given an open, writable file file, truncates the contents of the file so the current position in the file becomes the end of the file.

If you've wrapped the given file in some kind of buffering stream, be sure to flush it before calling this!

Params:
file

should be an instance of stream.NativeStream or some class like it which has a hidden field named "native". This field should contain a nativeobj which is the system-dependent file handle. This handle should obviously be for a file and not some other kind of object!

Throws:
ValueError

if file has no hidden field "handle", or if the "handle" hidden field does not name a valid file handle.

TypeError

if the "handle" hidden field is not a nativeobj.

IOException

if file could not be truncated for some reason (either because it was invalid or for some other reason).

global function writeTextFile(name: string, data: string, encoding: string = utf-8, errors: string = strict)

Writes the contents of the string data to the file name, encoding it with encoding.

If a file named name already exists, it will be replaced, and if it doesn't, it will be created. This is just like the "c" mode for outFile.

Params:
name

is the name of the file to write.

data

is the string which will be written out as the file's data.

encoding

is the text encoding to use, as found in the text library.

errors

is how malformed text should be read, as specified by the text library.

global function listDir(path: string, listHidden: bool, cb: function)

List the contents of the directory given by path, caling cb once for each entry in the directory.

Params:
path

is the path to the directory to list.

listHidden

controls whether or not hidden and system files are included in the listing.

cb

is the callback function which will be called once for each directory entry. It will be passed two parameters: the first is the name of the entry, and the second is a string indicating what kind of entry it is. This string will be one of "file" for regular files, "dir" for directories, "link" for file links, and "other" for other kinds of files (such as devices, special files, etc.).

This function can also optionally return a boolean false to halt the directory listing.

Throws:
OSException

if there was some kind of error listing the directory.

global function getDirListing(path: string, listHidden: bool, kinds: string)

An alternative method of getting a directory listing, this returns the directory's contents as an array of strings.

Params:
path

is the path to the directory to list.

listHidden

controls whether or not hidden and system files are included in the listing.

kinds

is a string which controls which kinds of directory entries are included in the resulting array. It must contain some combination of one or more of the following characters:

'f'

means regular files will be included.

'd'

means directories will be included. Their names will have a trailing slash ('/').

'l'

means links will be included. They will look the same as normal filenames.

'o'

means other kinds of entries will be included. They will look the same as normal filenames.

Returns:

an array of strings of directory entry names.

Throws:
ValueError

if kinds doesn't contain at least one of the specified characters.

global function exists(path: string)

Returns:

a bool saying whether or not there is an accessible file or directory named path. The key word here is "accessible". If you try to test the existence of a file you don't have permissions to access, this function will return false rather than throw an exception.

global function attrs(path: string, tab: table = null)

Get info on a file or directory at path and return it as a table.

Params:
path

is the file or directory you're interested in.

tab

is an optional table which will be filled with the attributes. If you pass nothing to this parameter, this function will create a new table and return that. This parameter is provided so you can get the attributes of several files in a row without having to allocate a new table each time.

Returns:

either a new table or tab if passed. The table will have the following fields:

"type"

is a string indicating whether it's a file ("file"), directory ("dir"), link ("link"), or something else ("other").

"size"

is an integer representing the size, in bytes, of the file. Meaningless for non-files.

"created"

is an integer representing the time of creation as measured in microseconds since midnight on January 1, 1970.

"modified"

is an integer representing the last time it was modified, measured the same way.

"accessed"

is an integer representing the last time it was accessed, measured the same way.

The three time fields are in a format compatible with that used by the time library.

Throws:
OSException

if the attributes could not be retrieved.

global function copyFromTo(from: string, to: string, force: bool = false)

Copies a file named by from to the path named by to. The function is named this way to remind you of the order of the arguments.

If a file named to already exists, the force parameter determines what happens. If force is false (the default), an exception is thrown. If force is true, the file at to is replaced by a copy of the file at from.

Throws:
OSException

if from could not be copied to to.

global function moveFromTo(from: string, to: string, force: bool = false)

Moves a file or directory named by from to the path named by to. The function is named this way to remind you of the order of the arguments.

This function lets you move directories as well as files, and you can also use it to rename files and directories (by simply moving them to another name in the same parent directory).

If a file or directory named to already exists, the force parameter determines what happens. If force is false (the default), an exception is thrown. If force is true, the file or directory at to is replaced by the file or directory at from.

Throws:
OSException

if from could not be moved to to.

global function remove(path: string)

Removes the file or directory at path entirely. This is an irreversible operation, so be careful!

If path is a directory, it must be empty.

Throws:
OSException

if path could not be removed.
