module hash

This library contains functionality common to both tables and namespaces, which are two similar kinds of hash tables. It also defines the opApply methods for tables and namespaces.

global function dup(t: table)

Makes a shallow duplicate of a table.

Returns:

a new table that has the same key-value pairs as t.

global function keys(h: table|namespace)

Returns:

an array containing all the keys of the given table or namespace. The order of the keys is arbitrary.

Notes:

This function is nontrivial, involving a memory allocation and a traversal of the given hashtable.

global function values(h: table|namespace)

Returns:

an array containing all the values of the given table or namespace. The order of the values is arbitrary.

Notes:

This function is nontrivial, involving a memory allocation and a traversal of the given hashtable.

global function apply(t: table, f: function)

Similar to the array.apply function, this iterates over the values of the table, calling the function f on each value, and storing the value it returns back into the same key. This works in-place on the table.

Examples:

If you have a table t = {x = 1, y = 2, z = 3} and call hash.apply(t, \x -> x + 5), t will now contain {x = 6, y = 7, z = 8}.

Returns:

t.

Throws:
TypeError

if f returns null.

global function map(t: table, f: function)

Like apply, but instead of operating in-place, creates a new table which holds the transformed key-value pairs, leaving t unmodified.

Returns:

the new table.

Throws:
TypeError

if f returns null.

global function reduce(t: table, f: function, initial = null)

Works just like the array.reduce function, but the order of the values is arbitrary. f will be called with two parameters, the current value of the accumulator and a new value from the table, and is expected to return the new value of the accumulator.

Just like array.reduce, the initial parameter can be given to set the accumulator to an initial value.

Throws:
ParamError

if you call this function on an empty table and don't pass an initial parameter.

global function filter(t: table, f: function)

Similar to the array.filter function, this creates a new table which holds only those key-value pairs for which the given filter function f returns true. f is given two arguments, the key and the value, and must return a boolean value. true means the key-value pair will be included in the result.

Examples:

hash.filter({a = 1, b = 2, c = "hi", d = 4.5, e = 6}, \k, v -> isInt(v)) will give a table containing only those key-value pairs from the original where the values were integers, so you will get {a = 1, b = 2, e = 6}.

Returns:

the new table.

Throws:
TypeError

if f returns anything other than a boolean value.

global function take(h: table|namespace)

Returns:

an arbitrary key-value pair as two values (first the key, then the value). The key-value pair is not removed. This is often useful when using a hashtable as a set.

Throws:
ValueError

if #h == 0.

global function pop(h: table|namespace)

Similar to take, but it does remove the key-value pair that is returned.

Returns:

the arbitrary key-value pair as two values (first the key, then the value).

Throws:
ValueError

if #h == 0.

global function clear(h: table|namespace)

Removes all key-value pairs from h.

global function remove(h: table|namespace, key)

Removes from h the key-value pair with key key, if any exists. For tables, you can also do this by assigning a null value to a key-value pair, but this function is the only way to remove key-value pairs from namespaces.

global function newNamespace(name: string, parent: namespace|null)

Create a new namespace.

This is useful for when you need to create a namespace with a programmatically-generated name, something that the language grammar doesn't allow.

This function behaves a little strangely with regards to its parameters, in order to mirror the behavior of actual namespace declarations.

Examples: 

hash.newNamespace("A") // the same as "namespace A {}"; the parent is set to the current function's environment.
hash.newNamespace("B", null) // the same as "namespace B : null {}"; B has no parent.
hash.newNamespace("C", _G) // the same as "namespace C : _G {}"; C's parent is the global namespace.

Params:
name

is the name of the new namespace.

parent

is the optional parent. There is a difference between passing null and passing nothing to this parameter, which goes against the usual behavior of optional parameters, but it's like this to mirror the behavior of real namespace declarations. See the examples for details.

Returns:

the new namespace.

function table.opApply(mode: string = null)

Iterate over the key-value pairs of a table.

Params:
mode

is the optional iteration mode. If you don't pass anything for this, attempting to insert or remove key-value pairs in the table while inside the foreach loop can cause strange behavior. If you want to insert or remove key-value pairs during iteration, pass the string "modify" for this parameter; this will iterate over the keys which existed at the time the loop started (as long as they still exist by the time iteration gets to them). Note that the "modify" iteration mode requires allocating an array at the beginning of iteration to hold the keys.

class WeakTableBase

Base class for all types of weak tables.

All the weak table classes present an interface as similar to actual tables as possible.

WeakTableBase.this()

All subclasses of this class must call this constructor with super().

class WeakKeyTable : WeakTableBase

A table with weak keys and strong values. This kind of table is often useful for associating data with objects, using the object as the key and the data you want to associate as the value.

Warnings:

It is possible for memory leaks to occur with this kind of table! Even though the keys are weak, the values may directly or indirectly reference the key objects, which means those objects can be kept alive even if the only thing that references them is a table like this. Be careful.

function WeakKeyTable.opApply(_)

Allows you to use a foreach loop over the table. You will not get weakref objects for the keys, but rather strong references. Also, you can modify the table during iteration.

function WeakKeyTable.each(f: function)

An alternate way of iterating over the table that doesn't incur any allocations. This will call f for each key-value pair in the table. You will not get weakref objects for the keys, but rather strong references. Modifying the table during iteration will cause erratic behavior.

Params:
f

is a function that will take two parameters, the key and the value, and can optionally return a true value to stop iteration.

Gets an array of the values of this table.

function WeakKeyTable.normalize()

Normalizes the table by removing any key-value pairs where the key has been collected. This is usually called automatically for you.
