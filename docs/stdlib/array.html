module array

The array library provides functionality for creating and manipulating arrays.

global function new(size: int, fill = null)

Creates an array object of length size, filling it with the value fill (which defaults to null).

Throws:
RangeError

if size is invalid.

global function new2D(size1: int, size2: int, fill = null)

Just like new, but creates an array of arrays. The outer array will have length size1, and each of its elements will be an array of length size2. Each of the sub-arrays will be filled with fill.

Throws:
RangeError

if size1 or size2 is invalid.

global function new3D(size1: int, size2: int, size3: int, fill = null)

Just like new2D, but creates an array of arrays of arrays. The outer array has length size1; its elements will be arrays of length size2; and those arrays' elements will be arrays of length size3, all filled with fill.

Throws:
RangeError

if size1, size2, or size3 is invalid.

global function range(val1: int, val2: int = null, step: int = null)

Creates a new array whose elements are a range of integers.

If only one argument is given, that argument specifies the noninclusive ending index, and the beginning index is assumed to be 0 and the step to be 1. This means array.range(5) will return [0, 1, 2, 3, 4] and array.range(-5) will return [0, -1, -2, -3, -4].

If two arguments are given, the first is the beginning inclusive index and the second is the ending noninclusive index. The step is again assumed to be 1. Examples: array.range(3, 8) gives [3, 4, 5, 6, 7]; array.range(2, -2) gives [2, 1, 0, -1]; and array.range(-10, -7) gives [-10, -9, -8].

Lastly, if three arguments are given, the first is the beginning inclusive index, the second the ending noninclusive index, and the third the step value. The step must be greater than 0; this function will automatically figure out that it needs to subtract the step if the ending index is less than the beginning index. Example: array.range(1, 20, 4) yields [1, 5, 9, 13, 17] and array.range(10, 0, 2) yields [10, 8, 6, 4, 2].

Throws:
RangeError

if step <= 0 or if the resulting array would be too large.

namespace array

This is the method namespace for array objects.

function array.opEquals(other: array)

Compares two arrays for shallow equality.

Shallow equality means two arrays are equal if they are the same length, and for each index i, this[i] is other[i] is true. This does not call opEquals metamethods on any of this arrays' elements.

function array.sort(how: function|string = null)

Sorts this array in-place. This is not a stable sort. This implementation uses smoothsort, which gives best- case linear time, and average-case and worst-case O(n log n) time, using constant space.

All the elements must be comparable with one another, and any opCmp metamethods will be called on the elements.

Params:
how

indicates how to sort this array:

  • If no value is given for this parameter, this array will be sorted in ascending order.

  • If the string "reverse" is given, this array will be sorted in descending order.

  • Otherwise, this parameter must be a function which is treated as a sorting predicate. It should take two arguments (which will be elements from this array), compare them, and return a comparison value (i.e. negative int if the first is less than the second, positive int if greater, and 0 if equal).

Returns:

this array.

Throws:
TypeError

if how was a function and it returned something other than an int.

function array.reverse()

Reverses this array's elements in place.

Returns:

this array.

function array.dup()

Returns:

a shallow copy of this array. Only the elements are copied, not any data that they point to.

function array.expand()

Returns:

all the elements of this array in order. In this way, you can "unpack" an array's values to pass as separate parameters to a function, or as return values, etc.

Throws:
ValueError

if this array is longer than 50 elements. Trying to return so many values can be a memory problem (and usually indicates a bug).

function array.toString()

Returns a nice string representation of this array. This will format this array into a string that looks like a Croc expression, like "[1, 2, 3]". String elements will also be surrounded by double quotes.

Note that the elements of the array do not have their toString metamethods called, since that could lead to infinite loops if this array references itself directly or indirectly. To get a more complete representation of an array, look at the dumpVal function (though that only outputs to the console).

function array.apply(func: function)

Iterates over this array, calling the function with each element of this array, and assigns the result of the function back into the corresponding array element.

Examples:

"[1, 2, 3, 4, 5].apply(\x -> x * x)" will replace the values in the array with "[1, 4, 9, 16, 25]".

Params:
func

should take one value and return one value.

Returns:

this array.

function array.map(func: function)

Same as apply, except this array is unmodified and the values returned by func are put into a new array of the same length.

Returns:

the new array.

function array.reduce(func: function, start = null)

Also known as fold or foldl (left fold) in many functional languages. This function takes a function func of two arguments which is expected to return a value. It treats this array as a list of operands, and uses func as if it were a left-associative binary operator between each pair of items in this array.

This sounds confusing, but it makes sense with a bit of illustration: "[1 2 3 4 5].reduce(\a, b -> a + b)" will sum all the elements of the array and return 15, since it's like writing ((((1 + 2) + 3) + 4) + 5). Notice that the operations are always performed left-to-right.

This function optionally takes a "start value" which will be used as the very first item in the sequence. For instance, "[1 2 3].reduce(\a, b -> a + b, 10)" will do the same thing as (((10 + 1) + 2) + 3). In the event that the array's length is 0, the start value is simply returned as is.

Returns:

the calculated value.

Throws:
ParamError

if #this == 0 and no value was passed for start.

function array.rreduce(func: function, start = null)

Similar to reduce, but treats func as a right-associative operator, meaning it goes through this array's elements in reverse order.

"[1 2 3 4 5].rreduce(\a, b -> a + b)" will still sum all the elements, because addition is commutative, but the order in which this is done becomes (1 + (2 + (3 + (4 + 5)))). Obviously if func is not commutative, reduce and rreduce will give different results.

Params:
func

start

is treated as an optional last value, which means "[1 2 3 4 5].rreduce(\a, b -> a + b), 10" is like writing (1 + (2 + (3 + (4 + (5 + 10))))).

Returns:

the calculated value.

Throws:
ParamError

if #this == 0 and no value was passed for start.

function array.filter(func: function)

Creates a new array which holds only those elements for which the given function returned true when called with elements from the source array.

The function is passed two arguments, the index and the value, and should return a boolean value. true means the given element should be included in the result, and false means it should be skipped.

"[1, 2, "hi", 4.5, 6].filter(\i, v -> isInt(v))" would result in the array "[1, 2, 6]", as the filter function only returns true for integral elements.

Returns:

the new array.

Throws:
TypeError

if func returns anything other than a bool.

function array.find(value, start: int = 0)

Performs a linear search for value in this array, starting at start and going right.

This works by looping over this array's elements, and if the element is the same type as value, it is compared (calling opCmp if necessary). The index of the first element that is the same type as value and which compares equal is returned.

This differs from "val in a" in that 'in' only checks if val is identical to any of the values in a; it never calls opCmp metamethods like this function does.

Params:
value

is the value to search for.

start

is the index where the search should begin. Can be negative to mean from the end of the array.

Returns:

#this if it wasn't found, or the index if it was.

Throws:
BoundsError

if start is invalid.

function array.findIf(pred: function, start: int = 0)

Performs a linear search starting at start and going right for the first element which, when passed to pred, causes pred to return true. This is a generic form of find.

Params:
pred

is the predicate function; it will be called with a single parameter (a value from this array) and should return a bool saying whether or not it is the value being searched for.

start

is the index where the search should begin. Can be negative to mean from the end of the array.

Returns:

#this if nothing was found (pred never returned true for any element), or the index if something was.

Throws:
BoundsError

if start is invalid.

TypeError

if pred returns something other than a bool.

function array.bsearch(value)

Performs a binary search for the value in this array. The array must be sorted for this search to work properly. Additionally, all the elements must be comparable (they had to be for the sort to work in the first place).

Returns:

#this if the value wasn't found, or its index if it was.

function array.pop(index: int = -1)

Removes a single element from this array (by default the last one), shifting up any elements after it if there are any, and returns the removed value.

This function makes it easy to use an array as a stack. Simply append values to push, and call a.pop() to pop.

Params:
index

is the index of the element to be removed. Can be negative to mean from the end of this array.

Returns:

the removed value.

Throws:
ValueError

if #this == 0.

BoundsError

if index is invalid.

function array.insert(index: int, value)

The inverse of pop, this inserts a value into this array at a given index, shifting down any elements after if if there are any.

Params:
index

is the index where value will be inserted. The value is inserted before the given index. index can also be #this, in which case the new value is simply appended to the end of this array. This can be negative to mean from the end of this array.

value

is the value to insert.

Returns:

this array.

Throws:
BoundsError

if index is invalid.

function array.swap(idx1: int, idx2: int)

Swaps the values at the given indices.

Both idx1 and idx2 can be negative to mean from the end of this array. If idx1 == idx2, this method is a no-op.

Returns:

this array.

Throws:
BoundsError

if idx1 or idx2 is invalid.

function array.set(vararg)

Something like the inverse of expand, this takes a variadic number of arguments, sets this array's length to the number of arguments passed, and copies the arguments into this array.

This is similar to using an array constructor, but reuses an array instead of allocating a new one.

Returns:

this array.

function array.max()

function array.extreme(pred: function)

A generic version of min and max, this uses a predicate function to determine which element is the most "extreme." If this

Params:
pred

should take two parameters: the first is the new value to be tested, and the second is the current extreme so far. pred should return true if the new value is more extreme than the previous and false otherwise.

To illustrate, a.extreme(\new, extreme -> new > extreme) will do the same thing as a.max() since the predicate returns true if the new value is bigger than the previous extreme. (In this case, using max is faster since it's optimized to do this, but this just illustrates the point.)

Returns:

the most extreme value.

Throws:
ValueError

if this array is empty.

TypeError

if pred returns anything other than a bool.

function array.any(pred: function = null)

This is a generalized boolean "or" (logical disjunction) operation.

Params:
pred

is an optional predicate function.

If none is passed, this method returns true if any element in the array has a truth value of true, and false otherwise.

If a function is passed, the function must take one parameter and return any value. The value returned from pred can be any type, only its truth value matters. This method will return true if pred returned a value with a truth value of true for any element in the array, and false otherwise.

Returns:

false if called on an empty array.

function array.all(pred: function = null)

This is a generalized boolean "and" (logical conjunction) operation.

Params:
pred

is an optional predicate function.

If none is passed, this method returns true if all the elements in the array have a truth value of true, and false otherwise.

If a function is passed, the function must take one parameter and return any value. The value returned from pred can be any type, only its truth value matters. This method will return true if pred returned a value with a truth value of true for all the elements in the array, and false otherwise.

Returns:

true if called on an empty array.

function array.fill(value)

Sets every element in the array to the given value.

function array.append(vararg)

Appends all the arguments to the end of the array, in order. This is different from the append operator (~=), because arrays will be appended as a single value, instead of having their elements appended.

function array.count(value, pred: function = null)

Counts the number of times value appears in this array, optionally using a predicate function to perform the comparison.

Params:
value

is the value to count.

pred

is the optional comparison predicate.

If pred is null, then this function simply loops over the array, testing if each element is equal to value (calling opCmp metamethods if needed), and counting each one which compares equal.

If pred is a function, it should take two values; the first will be elements from the array and the second will always be value. It should return a bool indicating whether the two values compare equal.

You can use this method instead of countIf to avoid creating a function closure, or to refactor code to use a cacheable function literal instead of a non-cacheable one.

Returns:

the number of elements which compared equal to value according to the behavior explained above.

Throws:
TypeError

if pred is a function and it returns anything other than a bool.

function array.countIf(pred: function)

Very similar to count, but more general. This version simply counts the number of items for which pred returns true.

Returns:

that count.

Throws:
TypeError

if pred returns anything other than a bool.

function array.opApply(mode: string = null)

This allows you to iterate over arrays using foreach loops.

foreach(i, v; a)
// ...

foreach(i, v; a, "reverse")
// iterate backwards
As the second example shows, passing in the string "reverse" as the second parameter will cause the iteration to run in reverse.

function array.flatten()

Flattens a multi-dimensional array into a single-dimensional array.

The dimensions can be nested arbitrarily deep. Always returns a new array. Can be called on single-dimensional arrays too, in which case it just returns a duplicate of the array.

Throws:
ValueError

if any array is directly or indirectly circularly referenced.
