The base library is a set of functions dealing with some language aspects which aren't covered by the syntax of the language, as well as miscellaneous functions that don't really fit anywhere else. The base library is always loaded when you create an instance of the Croc VM.

global function weakref(obj)

This function is used to create weak reference objects. If the given object is a value type (null, bool, int, or float), it simply returns them as-is. Otherwise returns a weak reference object that refers to the object. For each object, there will be exactly one weak reference object that refers to it. This means that if two objects are identical, their weak references will be identical and vice versa.

global function deref(obj: null|bool|int|float|weakref)

The parameter types for this might look a bit odd, but it's because this function acts as the inverse of weakref. If you pass a value type into the function, it will return it as-is. Otherwise, it will dereference the weak reference and return that object. If the object that the weak reference referred to has been collected, it will return null.

global function typeof(value)

This will get the type of the passed-in value and return it as a string. Possible return values are "null", "bool", "int", "float", "string", "table", "array", "function", "class", "instance", "namespace", "thread", "nativeobj", "weakref", and "funcdef".

global function niceTypeof(value)

This will get a more human-readable version of value's type and return it as a string. This is good for error messages and the like.

global function toString(value)

This is like rawToString, but it will call any toString metamethods defined for the value. Arrays have a toString metamethod defined for them by default, and any toString methods defined for class instances will be used.

Note that ints and floats will be converted to strings with default formatting (base 10, etc). If you need more control over the string formatting of numbers, use the string.format method.

global function rawToString(value)

This returns a string representation of the given value depending on its type, as follows:

  • null: the string "null".

  • bool: "true" or "false".

  • int: The decimal representation of the number.

  • float: The decimal representation of the number, to about 7 digits of precision.

  • string: The string itself.

  • nativeobj: A string formatted as "nativeobj 0x00000000", where 0x00000000 is the address of the native object that it references.

  • namespace: A string formatted as "namespace <name>", where <name> is the hierarchical name of the namespace.

  • function: If the function is native code, a string formatted as "native function <name>"; if script code, a string formatted as "script function <name>(<location>)".

  • funcdef: A string formatted as "funcdef <name>(<location>)".

  • For all other types, a string formatted as "<type> 0x<address>", where <type> is the name of the type and <address> is the memory address of the object.

class Vector

Croc's built-in array type is fine for most tasks, but they're not very well-suited to high-speed number crunching. Memblocks give you a low-level memory buffer, but don't provide any data structuring. Vectors solve both these problems: they are dynamically-resizable strongly-typed single-dimensional arrays of numerical values built on top of memblocks.

There are ten possible types a Vector can hold. Each type has an associated "type code", which is just a string. The types and their type codes are as follows:

Type Code

Definition

i8

Signed 8-bit integer

i16

Signed 16-bit integer

i32

Signed 32-bit integer

i64

Signed 64-bit integer

u8

Unsigned 8-bit integer

u16

Unsigned 16-bit integer

u32

Unsigned 32-bit integer

u64

Unsigned 64-bit integer

f32

Single-precision IEEE 754 float

f64

Double-precision IEEE 754 float

These type codes are case-sensitive, so for example, passing "u8" to the constructor is legal, whereas "U8" is not.

A note on the "u64" type: Croc's int type is a signed 64-bit integer, which does not have the range to represent all possible values that an unsigned 64-bit integer can. So when dealing with "u64" Vectors, values larger than 263 - 1 will be represented as negative Croc integers. However, internally, all the operations on these Vectors will be performed according to unsigned integer rules. The toString method is also aware of this and will print the values correctly, and if you'd like to print out unsigned 64-bit integers yourself, you can use toString(val, 'u') from the base library.

A note on all types: for performance reasons, Vectors do not check the ranges of the values that are stored in them. For instance, if you assign an integer into a "u8" Vector, only the lowest 8 bits will be stored. Storing floats into "f32" Vectors will similarly round the value to the nearest representable single-precision value.

Finally, the underlying memblock can be retrieved and manipulated directly; however, changing its size must be done carefully. If the size is set to a byte length that is not an even multiple of the item size of the Vector, an exception will be thrown the next time a method is called on the Vector that uses that memblock.

All methods, unless otherwise documented, return the Vector object on which they were called.

function Vector.range(type: string, val1: int|float, val2: int|float = null, step: int|float = 1)

Creates a Vector whose values are a range of ascending or numbers, much like the array.range function.

If the type parameter is one of the integral types, the next three parameters must be ints; otherwise, they can be ints or floats.

If called with just val1, it specifies a noninclusive end index, with a start index of 0 and a step of 1. So Vector.range("i32", 5) gives "Vector(i32)[0, 1, 2, 3, 4]", and Vector.range("i32", -5) gives "Vector(i32)[0, -1, -2, -3, -4]".

If called with val1 and val2, val1 will be the inclusive start index, and val2 will be the noninclusive end index. The step will be 1.

The step, if specified, specifies how much each successive element should differ by. The sign is ignored, but the step may not be 0.

Params:
type

is a string containing one of the type codes.

val1

is either the end index or the start index as explained above.

val2

is the optional end index; if specified, makes val1 the start index.

step

is the optional step size.

Returns:

the new Vector.

Throws:
RangeError

if step is 0.

RangeError

if the resulting Vector would have too many elements to be represented.

function Vector.type(type: string = null)

Gets or sets the type of this Vector.

If called with no parameters, gets the type and returns it as a string.

If called with a parameter, it must be one of the type codes given above. The Vector's type will be set to the new type, but only if the Vector's byte length is an multiple of the new type's item size. That is, if you had a "u8" Vector of length 7, and tried to change its type to "u16", it would fail because 7 is not an even multiple of the size of a "u16" element, 2 bytes.

When the type is changed, the data is not affected at all. The existing bit patterns will simply be interpreted according to the new type.

Params:
type

is the new type if changing this Vector's type, or null if not.

Returns:

the current type of the Vector if type is null, or nothing otherwise.

Throws:
ValueError

if type is not a valid type code.

ValueError

if the byte size is not an even multiple of the new type's item size.

function Vector.copyRange(lo1: int = 0, hi1: int = #this, other: Vector, lo2: int = 0, hi2: int = lo2 + (hi - lo))

Copies a slice of another Vector into a slice of this one without creating an unnecessary temporary.

function Vector.fill(val: int|float|function|array|Vector)

A flexible way to fill a Vector with data.

This method never changes the Vector's size; it always works in-place, and all data in this Vector is replaced. The behavior of this method depends on the type of its val parameter.

Params:
val

is the value used to fill this Vector.

If this is an integral Vector, and val is an int, all items in this Vector will be set to val.

If this is a floating-point Vector, val can be an int or float, and all items in this Vector will be set to the float representation of val.

If val is a function, it should take an integer that is the index of the element, and should return one value the appropriate type which will be the value placed in that index. This function is called once for each element this Vector.

If val is an array, it should be the same length as this Vector, be single-dimensional, and all elements must be valid types for this Vector. The values will be assigned element-for-element into this Vector.

Lastly, if val is a Vector, it must be the same length and type, and the data will be copied from val into this Vector.

function Vector.opApply(mode: string = "")

Allows you to iterate over the contents of a Vector using a foreach loop.

This works just like the opApply defined for arrays. The indices in the loop will be the element index followed by the element value. You can iterate in reverse by passing the string value "reverse" as the mode argument. For example:

local v = Vector.range("i32", 1, 6)
foreach(i, val; v) write(val) // prints 12345
foreach(i, val; v, "reverse") write(val) // prints 54321

Params:
mode

is the iteration mode. If it's "reverse", the iteration will run backwards.
